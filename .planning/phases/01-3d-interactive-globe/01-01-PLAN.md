  <name>Task 2: Build Globe3D React component with country highlighting</name>
  <files>src/components/Globe3D.tsx</files>
  <action>
Create src/components/Globe3D.tsx as a React component that renders an interactive 3D globe with visited countries highlighted.

**Props interface:**
```typescript
interface Globe3DProps {
  countries: Array<{ code: string; name: string }>;
  cities: Array<{ city: string; country: string; lat: number; lon: number; type: string; dates: string | null; note: string }>;
}
```

**Implementation details:**

1. **Data preparation (runs once via useMemo):**
   - Import `feature` from `topojson-client` and `countries` from `world-atlas/countries-110m.json`
   - Import `isoCountries` from `i18n-iso-countries` and register English locale
   - Convert visited country alpha-2 codes to numeric using `isoCountries.alpha2ToNumeric()` and store in a Set
   - Convert world-atlas topology to GeoJSON features via `feature(countries, countries.objects.countries)`
   - Map each GeoJSON feature to include a `visited` boolean by checking if `geo.id` is in the visitedCodes Set
   - Note: Vatican City (VA) may not have a matching polygon in countries-110m -- this is acceptable

2. **Globe rendering:**
   - Use `react-globe.gl` default Globe component with a ref
   - Set dimensions responsively: use a container div with ref, measure width on mount and resize via ResizeObserver, set height to width * 0.75 (maintain 4:3 aspect ratio)
   - Globe texture: use `"//unpkg.com/three-globe/example/img/earth-dark.jpg"` for dark theme consistency (NOT blue-marble -- site is dark mode)
   - Background: use `"//unpkg.com/three-globe/example/img/night-sky.png"` for star background
   - Enable atmosphere: `showAtmosphere={true}`, `atmosphereColor="rgba(99, 102, 241, 0.3)"` (indigo tint to match site primary), `atmosphereAltitude={0.15}`

3. **Country polygon styling (matches existing TravelMap color palette):**
   - `polygonCapColor`: visited countries = `'rgba(99, 102, 241, 0.6)'` (indigo, matching --map-visited-soft), unvisited = `'rgba(23, 23, 23, 0.6)'` (matching --map-unvisited)
   - `polygonSideColor`: `'rgba(50, 50, 50, 0.15)'`
   - `polygonStrokeColor`: `'rgba(255, 255, 255, 0.15)'` (subtle borders)
   - `polygonAltitude`: visited = 0.01, unvisited = 0.005 (slight elevation for visited)

4. **Interaction (built-in to react-globe.gl):**
   - Drag-to-rotate and scroll-to-zoom are automatic via Three.js OrbitControls (GLOBE-02)
   - Enable auto-rotate with slow speed: `autoRotate={true}`, `autoRotateSpeed={0.4}`
   - Auto-rotate pauses when user interacts (built-in behavior)

5. **Initial camera position:**
   - On mount, use `globeRef.current.pointOfView({ lat: 20, lng: -40, altitude: 2.5 })` to center on Atlantic (shows Americas + Europe)

6. **Performance (GLOBE-07):**
   - Component will be loaded via `client:only="react"` in Astro (prevents SSR, per research Pitfall 7)
   - Wrap the Globe import with a dynamic import check: if `typeof window === 'undefined'` return null (safety net)
   - Add a loading state: show a dark placeholder div with "Loading globe..." text while the globe initializes

7. **Container styling:**
   - Wrapper div: `position: relative`, `width: 100%`, `border-radius: 16px`, `overflow: hidden`
   - Match the existing .map-shell aesthetic: dark background with subtle border
   - Add `min-height: 450px` to prevent layout shift before globe loads
