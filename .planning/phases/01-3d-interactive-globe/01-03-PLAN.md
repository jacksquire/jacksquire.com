**Part A: Create src/components/GlobeToggle.astro**

This Astro component wraps both Globe3D and TravelMap, providing a toggle between them.

1. **Frontmatter:**
   - Import Globe3D from './Globe3D' (React component)
   - Import TravelMap from './TravelMap.astro' (existing flat map)
   - Import travel from '../data/travel.json'

2. **Template structure:**
   ```
   div.globe-toggle-container[data-globe-toggle]
     div.toggle-controls
       button[data-toggle-view].toggle-button
         span[data-globe-label] "Switch to Flat Map"
         span[data-map-label].hidden "Switch to 3D Globe"
     div[data-globe-view]
       Globe3D client:only="react"
         countries={travel.countriesVisited}
         cities={travel.citiesLived}
         countryNotes={travel.countryNotes}
     div[data-map-view].hidden
       TravelMap
   ```

   IMPORTANT: Use `client:only="react"` (NOT client:visible) to prevent SSR of Three.js (Pitfall 7 from research). The globe component handles its own loading state internally.

3. **Toggle script (is:inline for Astro):**
   - Query container, toggle button, globe/map views, label spans
   - Track isGlobeActive boolean (starts true)
   - On click: flip boolean, toggle 'hidden' class on globeView/mapView and label spans
   - Use classList.toggle(className, force) pattern for clean toggling

4. **Styling (scoped style block):**
   - `.globe-toggle-container`: position relative
   - `.toggle-controls`: position absolute, top 1rem, left 1rem, z-index 10
   - `.toggle-button`: Match existing `.map-control` style from TravelMap.astro:
     - `background: rgba(20, 20, 20, 0.85)`
     - `border: 1px solid rgba(255, 255, 255, 0.08)`
     - `color: var(--color-text)`
     - `font-size: 0.75rem`
     - `padding: 0.45rem 0.75rem`
     - `border-radius: 999px`
     - `cursor: pointer`
     - `transition: border-color 0.2s ease, transform 0.2s ease`

   - `.hidden`: `display: none !important`
